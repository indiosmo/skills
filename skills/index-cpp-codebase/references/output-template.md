# Output template

Worked examples of `INDEX.md` and `INDEX-report.md` against a generic
domain-driven C++ / CMake project. Read this when drafting a section so
the target shape is in front of you.

The example project (`acme-pricer`) is a multi-threaded UDP-driven
pricing service with a domain core, an inbound boundary that parses
network messages, an outbound boundary that emits CSV records, and a
thin runtime shell that wires sockets, threads, and event loops. The
shape is representative: shared utility library + leaf input/output
domains + composition domain + runtime.

---

## INDEX.md (full example)

```markdown
# INDEX

## System overview

Multi-threaded UDP-driven pricing service that consumes quote requests
and publishes priced records to stdout. Single-binary deployment with a
domain core and a thin runtime shell that wires sockets, threads, and
event loops.

## Tech stack

- C++23
- CMake (presets: debug, release, asan, tsan)
- Boost.LEAF -- https://github.com/boostorg/leaf
- fmt -- https://github.com/fmtlib/fmt
- Catch2 (tests) -- https://github.com/catchorg/Catch2

## Directory structure

acme-pricer/
|-- src/
|   |-- common/
|   |   `-- common/            general-purpose vocabulary headers
|   |       `-- network/       UDP / socket helpers
|   |-- request_parser/        inbound boundary: parses UDP payloads
|   |-- record_writer/         outbound boundary: formats CSV records
|   |-- pricing_core/          composition domain: pricing logic
|   `-- app/                   runtime shell + main
`-- test/                      per-module Catch2 unit tests (not expanded)

Skipped: `_build/` (CMake output, regenerated by the build command).

## Modules

### common

Base path: `acme-pricer/src/common/common/`

General-purpose vocabulary: strong types, result/expected error
plumbing, fixed-width strings, inplace function objects, CSV split
helpers, and the event loop primitive shared by every runtime thread.
Pure synchronous code over value types -- no threading lives here.

Headers:

- (P) result.hpp           -- Boost.LEAF result alias and helpers; interior error vocabulary
- (P) expected.hpp         -- C++20 `expected` backport used at module boundaries
- (P) error.hpp            -- generic error codes and `make_leaf_error` factory
- (P) fixed_string.hpp     -- bounded inline string for hot-path text
- (P) strong_type.hpp      -- compile-time-typed wrapper over primitives
- (P) inplace_function.hpp -- non-allocating `std::function`-shaped storage for callbacks
- (P) match.hpp            -- variant dispatch helper over `std::visit`
- (P) split_fields.hpp     -- whitespace/CSV tokenizer over views
- (P) from_chars.hpp       -- `std::from_chars` wrapper returning `result<T>`
- (P) concurrent_queue.hpp -- SPSC queue used between event-loop threads
- (P) event_loop.hpp       -- single-thread reactor: timers, posted callables, fd readiness
- (P) network/udp_socket.hpp -- non-blocking UDP socket wrapper

### request_parser

Base path: `acme-pricer/src/request_parser/request_parser/`

Translates incoming UDP CSV payloads into strongly-typed request
variants (quote, cancel, modify). Owns the parsing surface, the
per-request validation, and the structured error taxonomy emitted on
malformed input.

Headers:

- (P) types.hpp            -- request-side vocabulary (client_id, symbol, side, qty, price)
- (P) requests.hpp         -- variant of `quote_request`, `cancel_request`, `modify_request`
- (P) parser.hpp           -- CSV record -> request variant or structured error
- (P) error_code.hpp       -- enumerated parse/validation error codes
- (P) errors.hpp           -- structured error payloads for the leaf-handler taxonomy
- (I) detail/field_parsers.hpp -- per-field `from_chars`/`from_view` helpers

### record_writer

Base path: `acme-pricer/src/record_writer/record_writer/`

Translates domain events into CSV records and emits them to a sink
(stdout in production, a vector in tests). Owns the wire schema and
the event taxonomy (quote_ack, priced, book_update).

Headers:

- (P) types.hpp            -- event-side vocabulary (trade_id, price, qty)
- (P) events.hpp           -- variant of `quote_ack`, `priced`, `book_update`
- (P) writer.hpp           -- event -> CSV record formatter
- (P) sink.hpp             -- erased sink interface (stdout, vector, callback)

### pricing_core

Base path: `acme-pricer/src/pricing_core/pricing_core/`

Pricing domain. Matches incoming requests against resting orders and
produces priced events. Maintains per-instrument bid/ask ladders with
pool-backed intrusive nodes and a flat resting-request index.

Headers:

- (P) engine.hpp           -- pricing entry point; dispatches requests to handlers
- (P) types.hpp            -- domain vocabulary (book id, level, side)
- (I) v3/order_node.hpp    -- pool-backed intrusive node for resting requests
- (I) v3/book.hpp          -- per-instrument bid/ask ladders
- (I) v3/level.hpp         -- FIFO of resting requests at a single price level
- (I) v3/match_loop.hpp    -- core pricing algorithm

### app

Base path: `acme-pricer/src/app/app/`

Runtime shell. Owns `main`, the three-thread layout (source / core /
sink), the UDP source loop, the stdout sink loop, and the wiring that
connects them through `concurrent_queue`. The domain libraries are
threading-agnostic; threading lives here.

Headers:

- (P) app.hpp              -- application object; owns threads and queues
- (P) config.hpp           -- runtime configuration; parsed from argv/env
- (I) source_loop.hpp      -- UDP receive loop and parse pipeline
- (I) sink_loop.hpp        -- stdout flush loop
- (I) core_loop.hpp        -- pricing-core event loop
```

## INDEX-report.md (full example)

```markdown
# INDEX report

Companion to `INDEX.md`. Lists everything the run noticed but
deliberately left out of the index.

## Ambiguities

- `acme-pricer/src/common/common/network/udp_socket.hpp` -- only one
  external includer (`app/source_loop.cpp`). Tagged `(P)` per the bias
  rule; demote to `(I)` if the network helpers are meant to stay
  internal to the common library.

## Inconsistencies

- `acme-pricer/src/request_parser/README.md` describes a
  `validator.hpp` that no longer exists. The validation surface lives
  inside `parser.hpp` and the `errors.hpp` taxonomy. Updated
  description to match the code.

## Errors

- None.

## Decisions

- `common/network/` listed under the `common` module rather than as a
  separate module. Only one header, no separate README, shares the
  `common::` namespace.

## Skipped

- `_build/`, `cmake-build-debug/` -- CMake output.
- `docs/`, `*.md` at the repo root -- not source.
- `test/` -- listed once in the directory tree per the format rules.
```
